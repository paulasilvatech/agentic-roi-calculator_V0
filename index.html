Single Agent

One agent, focused tools, bounded scope. Use when the task fits in one prompt and one decision tree. Cheapest to build, easiest to govern, lowest ceiling. The right starting point for 70% of teams.Um agente, ferramentas focadas, escopo delimitado. Use quando a tarefa cabe em um prompt e uma árvore de decisão. Mais barato de construir, mais fácil de governar, teto mais baixo. O ponto de partida certo para 70% dos times.Un agente, herramientas enfocadas, alcance delimitado. Úsalo cuando la tarea cabe en un prompt y un árbol de decisión. Más barato de construir, más fácil de gobernar, techo más bajo. El punto de partida correcto para 70% de los equipos.

Best for: focused automation, well-defined tasks, narrow tooling.Ideal para: automação focada, tarefas bem definidas, tooling restrito.Ideal para: automatización enfocada, tareas bien definidas, tooling acotado.

Multi-Agent Pipeline

A handoff chain of specialized agents (planner, retriever, executor, reviewer). Use when the task has distinct stages with different model needs and a verifiable output between each stage. Higher cost, much higher ceiling, much harder to debug.Uma cadeia de handoff de agentes especializados (planner, retriever, executor, reviewer). Use quando a tarefa tem estágios distintos com necessidades de modelo diferentes e saída verificável entre cada um. Custo maior, teto muito maior, debug muito mais difícil.Una cadena de handoff de agentes especializados (planner, retriever, executor, reviewer). Úsalo cuando la tarea tiene etapas distintas con necesidades de modelo diferentes y salida verificable entre cada una. Costo mayor, techo mucho mayor, debug mucho más difícil.

Best for: complex workflows, mixed reasoning depth, audit-required steps.Ideal para: workflows complexos, raciocínio em níveis distintos, passos auditáveis.Ideal para: workflows complejos, razonamiento en niveles distintos, pasos auditables.

Pitfall 01Armadilha 01Trampa 01

Counting saved time as saved money.Contar tempo economizado como dinheiro economizado.Contar tiempo ahorrado como dinero ahorrado.

Productivity multiplied by salary is not a P&L line. The CFO never approves it. Convert to capacity reuse, error cost reduction, or revenue capacity, with a named conversion action.Produtividade vezes salário não é uma linha de P&L. O CFO nunca aprova. Converta em reuso de capacidade, redução de custo de erro ou capacidade de receita, com uma ação de conversão nomeada.Productividad por salario no es una línea de P&L. El CFO nunca aprueba. Convierte en reuso de capacidad, reducción de costo de error o capacidad de ingreso, con una acción de conversión nombrada.

Pitfall 02Armadilha 02Trampa 02

Ignoring HITL discount.Ignorar o desconto de HITL.Ignorar el descuento de HITL.

Human-in-the-loop review eats some of the productivity gain. Subtract it. Track it. Aim to reduce it through better controls, never by skipping review on irreversible actions.Revisão humana come parte do ganho. Subtraia. Acompanhe. Mire reduzir via melhores controles, nunca pulando revisão em ações irreversíveis.La revisión humana se come parte de la ganancia. Réstalo. Mídelo. Apunta a reducirlo vía mejores controles, nunca saltando revisión en acciones irreversibles.

Pitfall 03Armadilha 03Trampa 03

Skipping the eval harness.Pular o eval harness.Saltar el eval harness.

No eval set, no measurable quality, no defensible savings. The eval set is what lets you prove that the agent did not just shift the work to a different team. Build it before the first deploy.Sem eval set, sem qualidade mensurável, sem economia defensável. O eval set é o que prova que o agente não apenas transferiu o trabalho para outro time. Construa antes do primeiro deploy.Sin eval set, no hay calidad medible, no hay ahorro defendible. El eval set es lo que prueba que el agente no solo trasladó el trabajo a otro equipo. Constrúyelo antes del primer deploy.

Pitfall 04Armadilha 04Trampa 04

Picking a model before picking an archetype.Escolher modelo antes de escolher arquétipo.Elegir modelo antes de elegir arquetipo.

Model selection is the last decision, not the first. Pick the archetype, design the pipeline, then pick the model that fits per stage. Reverse the order and you optimize for the wrong cost line.Seleção de modelo é a última decisão, não a primeira. Escolha o arquétipo, desenhe o pipeline, depois escolha o modelo certo por estágio. Inverta a ordem e você otimiza para a linha de custo errada.La selección de modelo es la última decisión, no la primera. Elige el arquetipo, diseña el pipeline, luego elige el modelo correcto por etapa. Invierte el orden y optimizas para la línea de costo equivocada.

Pitfall 05Armadilha 05Trampa 05

Treating governance as a checkbox.Tratar governança como checkbox.Tratar gobernanza como checkbox.

Audit trail, IP boundary, model selection list, reversibility, skill review board. These are not slides, they are the difference between a pilot that scales and a pilot that gets quietly turned off after the security review.Audit trail, fronteira de IP, lista de modelos aprovados, reversibilidade, conselho de revisão de skills. Não são slides, são a diferença entre um piloto que escala e um piloto desligado em silêncio após a revisão de segurança.Audit trail, frontera de IP, lista de modelos aprobados, reversibilidad, consejo de revisión de skills.
